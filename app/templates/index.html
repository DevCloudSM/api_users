API User

Content of the user API database:

User database (Accessible by all API):
  • ID
  • Username
  • Name
  • Surname
  • Email

Purpose of the User API:

The User API aims to provide functionality that allows applications to manage user profiles, including creating, retrieving, updating, and deleting user information.

Main objectives of the User API:
  1. Create user: Allow new users to be registered by providing the necessary information such as: name; first name; pseudo; mail; id.
    Ideas (If there is time): address; country; departments (in the company); businesses; PP; descriptions

  2. Create users randomly.

  3. Delete User: Allow applications to delete a specific user from the database, if applicable.

  4. Retrieve user information: Provide a method to retrieve details of a specific user based on their unique ID or other search criteria.
    • Search function with different search attributes (Like last name, first name...)
  5. Update a user's information: Allow apps to update an existing user's information, such as name, email, password, etc.

  6. Permissions and roles management: Integrate features to manage user permissions and roles, for example allowing administrators to set user access privileges.
